搭建环境

VMware虚拟机
架构：x86_64
OS：银河麒麟桌面操作系统V10(SP1)
内核：5.10.0.8-generic

Alist的安装

Alist官网：https://alist.nn.ci/zh/
Alist github仓库：https://github.com/alist-org/alist
Alist docker仓库页面：https://registry.hub.docker.com/r/xhofe/alist
Alist可以通过4种方法进行安装部署，分别是一键脚本、手动安装、Docker容器(https://download.docker.com/linux/static/stable/x86_64/) 、桌面版
其中桌面版需要付费购买，我看了以下要9.99$ ，是美元 USD $
这里将记录一键脚本、手动安装、docker安装的过程

自定义路径安装

默认安装在 /opt/alist 中。 自定义安装路径，将安装路径作为第二个参数添加，必须是绝对路径（如果路径以 alist 结尾，则直接安装到给定路径，否则会安装在给定路径 alist 目录下），如 安装到 /root：

  • 启动: systemctl start alist
  • 关闭: systemctl stop alist
  • 状态: systemctl status alist
  • 重启: systemctl restart alist

获取密码

3.25.0以上版本将密码改成加密方式存储的hash值，无法直接反算出密码，如果忘记了密码只能通过重新 随机生成 或者 手动设置

获取 AList

打开 AList Release 从github下载待部署系统对应的文件。最新版的前端已经和后端打包好了，不用再下载前端文件了。
例如本次搭建环境为linux系统 x86架构，故选择下载 alist-linux-amd64.tar.gz

使用docker部署alist

离线安装docker

参考教程链接：https://blog.csdn.net/Rong_zhe/article/details/125402773

从https://download.docker.com/linux/ubuntu/dists/xenial/pool/stable/amd64/下载docker的安装包
这里的安装包共有3类，即 containerd.io、 docker-ce-cli、 docker-ce 三类安装包之间存在着依赖关系，所以安装顺序如下

1. 安装容器类服务，使用 dpkg -i containerd.io_XXXXXX.deb 指令进行安装。
2. 安装docker客户端，cli 即为 client 的意思。使用 dpkg -i docker-ce-cli_XXXX.deb 指令进行安装。
3. 安装docker服务端程序，使用 dpkg -i docker-ce_XXXX.deb 指令进行安装。
